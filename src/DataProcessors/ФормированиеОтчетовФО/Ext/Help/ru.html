Обработка "Формирование отчетов ФО".

Обработка служит для формирования документов "Отчет ФО" для выгрузки в товароучетную программу.

Вызывается из журнала документов "Отчеты ФО".

В форме указывается дата конца периода, за который формируются отчеты. Начальная дата определяется автоматически на основании последних сформированных отчетов. Таким образом, новые отчеты всегда формируются от даты последнего сформированного отчета, имеющегося в журнале, и до даты, указанной в диалоге.

В списке отмечаются флажками те места реализации, по которым нужно осуществить выгрузку.

После указания всех параметров, для начала формирования документов нажмите кнопку «Выполнить».

Если дата конца периода формирования отчетов, заданная в диалоге, окажется меньше или равной дате последнего сформированного отчета, то имеющиеся более поздние отчеты должны быть удалены, о чем будет выдано предупреждение.

В результате формирования, в журнале «Отчеты о продажах» появятся  новые документы вида «Отчет ФО». Количество этих новых документов зависит от количества выбранных мест реализации,   наличия списанных товаров, и алгоритма группировки. 

При автоматическом формировании документов «Отчет ФО», документы формируются не по дням, а по сменам. Так, если в пределах одного рабочего дня (или суток) в ресторане успевает отработать несколько смен, то и отчеты будут сформированы в пределах смен.

Кроме того, при формировании документов «Отчет ФО», информация о продажах группируется по следующему алгоритму, и по каждой группировке создается отдельный документ (или группа документов): 

·          по местам реализации,

·          по группам оплаты (если они используются),

·          по типу расхода (Реализация/Списание):

·          реализация – товары и номенклатурные специфики из заказов,

·          списание – товары и номенклатурные специфики из Удалений с признаком причины «Списывать продукты»,

·          по типам оплаты:

·          Наличными (не детализируется по Контрагентам, то есть Контрагент в документе не указывается),

·          Банковской картой (не детализируется по контрагентам),

·          На неплательщика (создается отдельный документ на каждого контрагента),

·          Безнал  (создается отдельный документ на каждого контрагента),

·          Смешанный тип оплаты: если есть оплата На неплательщика или Безнал, то   (создается отдельный документ на каждого контрагента), если нет ни На неплательщика, ни Безнал, то не детализируется по контрагентам (Контрагент в документе не указывается),

·          По Контрагентам (если был указан реквизит Клиента).

 Таким образом, в каждом из документов «Отчет ФО» может быть указано только одно место реализации, одна группа оплаты, один тип расхода, один тип оплаты и один контрагент.
